# Authoring Checklist

- [ ] `description` starts with `Use when` or intentionally uses `Use proactively` only when broad proactive triggering is required
- [ ] trigger text is task-based and concise
- [ ] section structure matches the actual skill instead of an inherited scaffold
- [ ] workflow is ordered and deterministic
- [ ] `scripts/validate-skill.sh` returns PASS
- [ ] optional helper scripts are documented only when they provide real value
