# Authoring Checklist

- [ ] `description` is aggressive and directive — uses `DO NOT <default behavior>` to block the agent's instinct, then mandates repomap as the FIRST action. Starts with `DO NOT` or `Always` — never passive "Use when"
- [ ] Every coding scenario is covered: entering a codebase, before editing, after editing
- [ ] Skip clause is narrow: only trivial single-line fixes or pure non-code tasks
- [ ] trigger text is task-based and concise
- [ ] section structure matches the actual skill instead of an inherited scaffold
- [ ] workflow is ordered and deterministic
- [ ] `scripts/validate-skill.sh` returns PASS
- [ ] optional helper scripts are documented only when they provide real value
